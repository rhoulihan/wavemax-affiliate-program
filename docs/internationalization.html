Overview

The WaveMAX Affiliate Program includes comprehensive internationalization support, enabling deployment in multiple languages and regions. The system automatically detects user language preferences and provides a localized experience throughout the application.

✅ Key Features:
  • Automatic browser language detection
  • Persistent language preferences in user profiles
  • Real-time language switching without page reload
  • Localized email communications
  • Parent-iframe language synchronization

Implementation Details

Client-Side Translation

The i18n system is implemented using a custom JavaScript library that provides real-time translation without page reloads.

Recent Enhancement: All UI components including the new swirl spinner loading animations, address validation modals, and geocoding confirmation dialogs now support full internationalization.

Language Detection

The system uses a hierarchical approach to determine the user's preferred language:

  1. Saved Preference - localStorage value from previous sessions
  2. URL Parameter - ?lang=es in the URL
  3. Browser Language - Navigator language API
  4. Default Language - English (en)

Registration Language Capture

// Automatically captures browser language during registration
const browserLang = navigator.language || navigator.userLanguage || 'en';
const langCode = browserLang.substring(0, 2).toLowerCase();
const supportedLanguages = ['en', 'es', 'pt', 'de'];
const languagePreference = supportedLanguages.includes(langCode) ? langCode : 'en';

// Stored in database models
{
    languagePreference: {
        type: String,
        enum: ['en', 'es', 'pt', 'de'],
        default: 'en'
    }
}

Email Template Localization

The email system has been fully internationalized with a placeholder-based template system that supports all four languages:

✅ New Email Translation System: All email templates now use a [PLACEHOLDER] system with complete translations for English, Spanish, Portuguese, and German.

Placeholder-Based Templates

Email templates use placeholders that are replaced with language-specific content at runtime:

<!-- Email template with placeholders -->
<h1>[EMAIL_HEADER]</h1>
<p>[GREETING]</p>
<p>[WELCOME_MESSAGE]</p>

Email Service Implementation

// Email service with full translation support
exports.sendAffiliateWelcomeEmail = async (affiliate) => {
    const language = affiliate.languagePreference || 'en';
    const template = await loadTemplate('affiliate-welcome', language);
    
    // Complete translations for all languages
    const translations = {
        en: {
            EMAIL_TITLE: 'Welcome to WaveMAX Laundry Affiliate Program',
            EMAIL_HEADER: 'Welcome to WaveMAX Laundry Affiliate Program',
            GREETING: `Hello ${affiliate.firstName},`,
            WELCOME_MESSAGE: 'Welcome to our affiliate program!'
            // ... all placeholders
        },
        es: {
            EMAIL_TITLE: 'Bienvenido al Programa de Afiliados de WaveMAX Laundry',
            // ... Spanish translations
        },
        pt: {
            EMAIL_TITLE: 'Bem-vindo ao Programa de Afiliados WaveMAX Laundry',
            // ... Portuguese translations
        },
        de: {
            EMAIL_TITLE: 'Willkommen beim WaveMAX Laundry Affiliate-Programm',
            // ... German translations
        }
    };
    
    // Replace placeholders with translated content
    const t = translations[language] || translations.en;
    let html = template;
    for (const [key, value] of Object.entries(t)) {
        html = html.replace(new RegExp(`\\[${key}\\]`, 'g'), value);
    }
    
    await sendEmail(affiliate.email, t.EMAIL_TITLE, html);
};

Usage Guide

For Developers

Adding Translatable Content

<!-- HTML -->
<button data-i18n="common.buttons.submit">Submit</button>

// JavaScript
const message = i18n.t('common.messages.success');
alert(message);

Using Parameters

<!-- HTML with parameters -->
<p data-i18n="welcome.greeting" 
   data-i18n-params='{"name": "John"}'>
   Welcome, John!
</p>

// JavaScript with parameters
const greeting = i18n.t('welcome.greeting', { name: 'John' });
// Output: "Welcome, John!" (or localized equivalent)

For End Users

  • First Visit: System automatically detects browser language
  • Language Selection: Use the language dropdown on any page
  • Persistence: Selection is saved for future visits
  • Emails: Received in the language selected during registration

Adding New Languages

Step-by-Step Guide to Add a New Language

  1. Update Supported Languages

    Edit /public/assets/js/i18n.js:

    supportedLanguages: ['en', 'es', 'pt', 'de', 'fr'], // Add 'fr' for French

  2. Create Translation File

    Create /public/locales/fr/common.json:

    {
    "landing": {
        "header": {
            "title": "Devenez Partenaire Affilié WaveMAX",
            "subtitle": "Gagnez des commissions en référant des clients"
        }
        // ... complete translations
    }
}

  3. Update Database Models

    Update enum in both Affiliate.js and Customer.js:

    languagePreference: {
    type: String,
    enum: ['en', 'es', 'pt', 'de', 'fr'], // Add 'fr'
    default: 'en'
}

  4. Add Email Templates

    Create email templates in /server/templates/emails/fr/

  5. Update Email Service

    Add translated subjects in emailService.js:

    const subjects = {
    en: 'Welcome to WaveMAX',
    es: 'Bienvenido a WaveMAX',
    pt: 'Bem-vindo ao WaveMAX',
    de: 'Willkommen bei WaveMAX',
    fr: 'Bienvenue chez WaveMAX'  // Add French
};

  6. Update Parent Bridge Script

    Add flag image sources for the new language if using dropdown integration

Best Practices

Translation Guidelines

  • Consistency: Use consistent terminology across all translations
  • Context: Consider cultural context, not just literal translation
  • Length: Account for text expansion (German ~30% longer than English)
  • Testing: Test UI with longest translations to ensure proper layout

UI Component Translations

  • Loading Indicators: Swirl spinner messages are fully translatable 
    // Example: Loading spinner with i18n
SwirlSpinnerUtils.showOnForm(form, {
    message: i18n.t('common.loading.submitting')
});
  • Address Validation: All geocoding and validation messages support i18n 
    // Confirmation modal with translated content
showModal(i18n.t('address.validation.confirm'), content);
  • Dynamic Content: Use data attributes for complex interpolations 
    <div data-i18n="pricing.preview" 
     data-i18n-params='{"bags": 5, "fee": "$45"}'></div>

Development Tips

  • Key Naming: Use descriptive, hierarchical keys (e.g., landing.header.title)
  • Fallbacks: Always provide English fallback for missing translations
  • Parameters: Use parameters for dynamic values instead of concatenation
  • HTML Safety: Escape user input to prevent XSS attacks

Common Pitfalls to Avoid

  • ❌ Don't put data-i18n on elements with dynamic children (IDs will be lost)
  • ❌ Don't hardcode language-specific text in JavaScript
  • ❌ Don't forget to update email templates when adding new languages
  • ❌ Don't assume text direction (prepare for RTL languages)
⚠️ Important Note: When using data-i18n attributes, the entire content of the element will be replaced. Do not use on parent elements that contain dynamic content or elements with IDs that JavaScript needs to reference.
