Implementation Details

Client-Side Translation

The i18n system is implemented using a custom JavaScript library that provides real-time translation without page reloads.

Recent Enhancement: All UI components including the new swirl spinner loading animations, address validation modals, and geocoding confirmation dialogs now support full internationalization.

Core Library

// Located at: /public/assets/js/i18n.js
window.i18n = {
    supportedLanguages: ['en', 'es', 'pt', 'de'],
    currentLanguage: 'en',
    translations: {},
    
    // Set language
    setLanguage: function(lang) {
        // Implementation
    },
    
    // Translate a key
    t: function(key, params) {
        // Implementation with parameter interpolation
    },
    
    // Get current language
    getCurrentLanguage: function() {
        return this.currentLanguage;
    }
};

Language Detection

The system uses a hierarchical approach to determine the user's preferred language:

  1. Saved Preference - localStorage value from previous sessions
  2. URL Parameter - ?lang=es in the URL
  3. Browser Language - Navigator language API
  4. Default Language - English (en)

Registration Language Capture

// Automatically captures browser language during registration
const browserLang = navigator.language || navigator.userLanguage || 'en';
const langCode = browserLang.substring(0, 2).toLowerCase();
const supportedLanguages = ['en', 'es', 'pt', 'de'];
const languagePreference = supportedLanguages.includes(langCode) ? langCode : 'en';

// Stored in database models
{
    languagePreference: {
        type: String,
        enum: ['en', 'es', 'pt', 'de'],
        default: 'en'
    }
}

Email Service Implementation

// Email service automatically uses user's language preference
exports.sendAffiliateWelcomeEmail = async (affiliate) => {
    const language = affiliate.languagePreference || 'en';
    const template = await loadTemplate('affiliate-welcome', language);
    
    // Translated subjects
    const subjects = {
        en: 'Welcome to WaveMAX Laundry Affiliate Program',
        es: 'Bienvenido al Programa de Afiliados de WaveMAX Laundry',
        pt: 'Bem-vindo ao Programa de Afiliados WaveMAX Laundry',
        de: 'Willkommen beim WaveMAX Laundry Affiliate-Programm'
    };
    
    await sendEmail(affiliate.email, subjects[language], html);
};

Usage Guide

For Developers

Adding Translatable Content

<!-- HTML -->
<button data-i18n="common.buttons.submit">Submit</button>

// JavaScript
const message = i18n.t('common.messages.success');
alert(message);

Using Parameters

<!-- HTML with parameters -->
<p data-i18n="welcome.greeting" 
   data-i18n-params='{"name": "John"}'>
   Welcome, John!
</p>

// JavaScript with parameters
const greeting = i18n.t('welcome.greeting', { name: 'John' });
// Output: "Welcome, John!" (or localized equivalent)

For End Users

  • First Visit: System automatically detects browser language
  • Language Selection: Use the language dropdown on any page
  • Persistence: Selection is saved for future visits
  • Emails: Received in the language selected during registration

Best Practices

Translation Guidelines

  • Consistency: Use consistent terminology across all translations
  • Context: Consider cultural context, not just literal translation
  • Length: Account for text expansion (German ~30% longer than English)
  • Testing: Test UI with longest translations to ensure proper layout

UI Component Translations

  • Loading Indicators: Swirl spinner messages are fully translatable 
    // Example: Loading spinner with i18n
SwirlSpinnerUtils.showOnForm(form, {
    message: i18n.t('common.loading.submitting')
});
  • Address Validation: All geocoding and validation messages support i18n 
    // Confirmation modal with translated content
showModal(i18n.t('address.validation.confirm'), content);
  • Dynamic Content: Use data attributes for complex interpolations 
    <div data-i18n="pricing.preview" 
     data-i18n-params='{"bags": 5, "fee": "$45"}'></div>

Development Tips

  • Key Naming: Use descriptive, hierarchical keys (e.g., landing.header.title)
  • Fallbacks: Always provide English fallback for missing translations
  • Parameters: Use parameters for dynamic values instead of concatenation
  • HTML Safety: Escape user input to prevent XSS attacks

Common Pitfalls to Avoid

  • ‚ùå Don't put data-i18n on elements with dynamic children (IDs will be lost)
  • ‚ùå Don't hardcode language-specific text in JavaScript
  • ‚ùå Don't forget to update email templates when adding new languages
  • ‚ùå Don't assume text direction (prepare for RTL languages)
