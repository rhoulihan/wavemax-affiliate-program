Sample Data Generation

🎯 Purpose: The sample data generation script creates realistic test data for the WaveMAX Affiliate Program, allowing developers and testers to work with a fully populated system without using production data.

Overview

The sample data generator creates a comprehensive test dataset that simulates real-world usage patterns. This tool is essential for:

  • Development and testing of new features
  • Performance testing with realistic data volumes
  • UI/UX testing with varied data scenarios
  • Training and demonstrations
  • QA testing of edge cases

What Gets Generated

Affiliates

20

Customers

200-400

Orders

1,400-4,800

Time Period

60 days

Data Characteristics

  • Affiliates: 20 affiliates with complete business profiles
  • Customers: 10-20 customers per affiliate (200-400 total)
  • Orders: 7-12 orders per customer over 60 days
  • Order Status: ~1/3 of customers have an active order
  • Processing Time: 20-30 hours for completed orders
  • Bags per Order: 1-2 bags averaging 30 lbs each
  • Location: All data centered around Austin, TX area

Quick Start

Basic Usage

# Generate sample data (preserves existing non-test data)
node scripts/generate-sample-data.js

# Clean existing test data before generating new
node scripts/generate-sample-data.js --clean
💡 Tip: The script only affects test data (emails ending with @example.com), so it's safe to run in development environments with existing data.

Generated Data Details

Affiliates

Each affiliate is created with:

  • Unique affiliate ID (AFF001, AFF002, etc.)
  • Complete business information
  • Service area centered in Austin, TX with 5-15 mile radius
  • Delivery fee structure ($10-15 minimum, $5-8 per bag)
  • Payment method (direct deposit, check, or PayPal)
  • Active and approved status

Customers

Each customer includes:

  • Unique customer ID with timestamp
  • Complete personal information
  • Austin, TX area address
  • Associated with one affiliate
  • Default payment method (credit card)
  • Special delivery instructions

Orders

Orders are distributed realistically:

  • In-Process Orders: ~33% of customers have one active order
  • Order Status: scheduled, processing, or processed
  • Completed Orders: Spread over the last 60 days
  • Pricing: $1.25/lb wash & fold + delivery fees
  • Commission: 10% of wash & fold total

Configuration Options

The script includes configurable parameters at the top of the file:

Parameter Default Value Description
NUM_AFFILIATES 20 Number of affiliates to create
MIN_CUSTOMERS_PER_AFFILIATE 10 Minimum customers per affiliate
MAX_CUSTOMERS_PER_AFFILIATE 20 Maximum customers per affiliate
MIN_ORDERS_PER_CUSTOMER 7 Minimum orders per customer
MAX_ORDERS_PER_CUSTOMER 12 Maximum orders per customer
DAYS_HISTORY 60 Days of order history to generate
AVG_LBS_PER_BAG 30 Average weight per laundry bag

Technical Implementation

Key Features

  • Consistent Seeding: Uses faker.seed() for reproducible data
  • Safe Cleanup: Only removes test data (emails ending in @example.com)
  • Realistic Patterns: Orders distributed over time with proper status progression
  • Geographic Accuracy: All addresses use real Austin, TX zip codes
  • Password Security: Uses proper PBKDF2 hashing with salt

Database Models Used

  • Administrator - For system configuration
  • Affiliate - Business partner accounts
  • Customer - End user accounts
  • Order - Laundry service orders
  • SystemConfig - System-wide settings

Testing Different Scenarios

The generator creates various scenarios for testing:

Order Status Distribution

  • Scheduled: Orders awaiting pickup
  • Processing: Orders being washed
  • Processed: Orders ready for delivery
  • Complete: Delivered orders with payment

Payment Methods

  • Affiliates: Direct deposit, check, or PayPal
  • Customers: Credit cards (Visa, Mastercard, Amex, Discover)

Performance Considerations

📊 Data Volume: Default settings generate approximately:
  • 20 affiliates
  • 300 customers (average)
  • 2,700 orders (average)
  • 5,400 bag records (average)

Generation time depends on your system but typically takes 30-60 seconds for the default configuration.

Troubleshooting

Common Issues:
  • MongoDB Connection: Ensure MongoDB is running and MONGODB_URI is set correctly
  • Memory Issues: For very large datasets, increase Node.js memory: node --max-old-space-size=4096 scripts/generate-sample-data.js
  • Duplicate IDs: The script uses timestamps in IDs to avoid duplicates

Best Practices

  • Run with --clean flag to ensure consistent test data
  • Use in development and staging environments only
  • Adjust configuration parameters for specific test scenarios
  • Monitor database size when generating large datasets
  • Document any custom modifications for team awareness
🚀 Pro Tip: Create multiple configuration presets by copying the script and adjusting parameters for different testing scenarios (e.g., high-volume testing, edge cases, specific date ranges).
