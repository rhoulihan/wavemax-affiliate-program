Administrator Setup Guide

‚ÑπÔ∏è Overview: This guide covers the complete process of setting up administrator accounts for the WaveMAX Affiliate Program, including initial setup, security configuration, and best practices.

Quick Start: Interactive Admin Creation Script

Step 1: Ensure Prerequisites

  • MongoDB is running and accessible
  • Environment variables are configured in .env
  • Node.js dependencies are installed
  • Email service configured for welcome emails

Step 2: Start the Server

The system automatically creates a default administrator account on first startup:

npm start
# or with PM2
pm2 start ecosystem.config.js

Default administrator credentials:

  • Email: Set by DEFAULT_ADMIN_EMAIL environment variable (defaults to admin@wavemaxlaundry.com)
  • Password: WaveMAX!2024
  • Permissions: All (super admin)
‚ö†Ô∏è Important: You will be required to change the password on first login.

Step 3: First Login and Password Change

Navigate to the administrator login page:

https://yourdomain.com/embed-app.html?route=/administrator-login
  1. Log in with the default credentials
  2. You will be immediately prompted to change your password
  3. Enter your current password (WaveMAX!2024)
  4. Enter a new password that meets these requirements:
    • At least 12 characters long
    • Contains uppercase and lowercase letters
    • Contains numbers and special characters
    • Not one of your last 5 passwords
  5. After changing your password, log in again with the new password

Step 4: Security Features

The system includes these security features:

  • Forced Password Change: Required on first login
  • Password History: Prevents reuse of last 5 passwords
  • Account Lockout: After 5 failed login attempts
  • Audit Logging: All authentication events are logged
  • Session Management: Secure JWT tokens with refresh capability
‚úÖ Enhanced Security: The automatic setup process ensures secure defaults with forced password change, password history tracking, and comprehensive audit logging.

Customizing the Default Administrator

To customize the default administrator account, modify init-admin.js before the first server startup:

// Create default administrator account
const defaultAdmin = new Administrator({
    firstName: 'System',
    lastName: 'Administrator',
    email: 'admin@yourcompany.com',           // Change default email
    password: 'YourSecurePassword123!',       // Change default password
    permissions: ['all'],                     // Adjust permissions if needed
    isActive: true,
    requirePasswordChange: true               // Always force password change
});
‚ÑπÔ∏è Note: The default administrator is only created if no administrators exist in the database. Existing installations are not affected.

Manual Administrator Creation via MongoDB

For direct database creation, use the MongoDB shell:

# Connect to MongoDB
mongosh mongodb://localhost:27017/wavemax

# Switch to the database
use wavemax

# Create administrator manually
db.administrators.insertOne({
    adminId: "ADM003",
    firstName: "Admin",
    lastName: "User",
    email: "admin@wavemaxlaundry.com",
    password: "$2b$10$hashedpasswordhere",  // Use bcrypt to hash
    permissions: [
        "system_config",
        "operator_management",
        "view_analytics",
        "manage_affiliates"
    ],
    isActive: true,
    loginAttempts: 0,
    createdBy: "manual",
    createdAt: new Date(),
    updatedAt: new Date()
})
‚ÑπÔ∏è Note: When creating administrators manually, you must hash the password using bcrypt with 10 rounds.

Creating Additional Administrators

After the initial setup, you can create additional administrators through the dashboard:

  1. Log in as an administrator with the administrators.create permission
  2. Navigate to the Administrator Management section
  3. Click "Create New Administrator"
  4. Fill in the required information:
    • First Name and Last Name
    • Email address (must be unique)
    • Initial password (user will be forced to change on first login)
    • Permissions (select from available options)
  5. The new administrator will receive a welcome email with login instructions
‚ÑπÔ∏è Note: All new administrators are required to change their password on first login for security purposes.

Available Administrator Permissions

First-Time Login Process

Step 1: Access the Administrator Login

Navigate to:

https://yourdomain.com/embed-app.html?route=/administrator-login

Or for local development:

http://localhost:3000/embed-app.html?route=/administrator-login

Step 2: Enter Credentials

  • Email: The email address used during setup
  • Password: The password set during creation

Step 3: Password Change (First Login)

When logging in with the default account for the first time:

  1. You will be automatically redirected to the password change form
  2. Enter your current password (WaveMAX!2024)
  3. Enter a new password that meets all requirements
  4. Confirm the new password
  5. Click "Change Password"
  6. Log in again with your new password
‚ö†Ô∏è Important: You cannot access any other features until the password is changed.

Password Requirements

Strong Password Requirements:
  • Minimum 12 characters
  • At least one uppercase letter
  • At least one lowercase letter
  • At least one number
  • At least one special character (!@#$%^&*)
  • No common dictionary words
  • Not similar to email or name
  • Cannot be any of your last 5 passwords

Account Recovery Options

Password Reset via Email

  1. Click "Forgot Password?" on the login page
  2. Enter the administrator email address
  3. Check email for reset instructions
  4. Click the reset link (valid for 1 hour)
  5. Set a new password

Manual Password Reset

If email is not available, reset via database:

# Generate new password hash
const bcrypt = require('bcrypt');
const newHash = await bcrypt.hash('NewSecurePassword123!', 10);

# Update in MongoDB
db.administrators.updateOne(
    { email: "admin@wavemaxlaundry.com" },
    { 
        $set: { 
            password: newHash,
            loginAttempts: 0,
            lastPasswordChange: new Date()
        }
    }
)

Security Best Practices

Administrator Account Security Checklist

  • Use unique, strong passwords for each administrator
  • Enable two-factor authentication when available
  • Regularly rotate passwords (every 90 days)
  • Limit administrator accounts to essential personnel only
  • Use principle of least privilege for permissions
  • Monitor administrator login activity
  • Disable accounts immediately when employees leave
  • Use separate accounts for daily operations vs. admin tasks
  • Never share administrator credentials
  • Always use HTTPS for administrator access

Multiple Administrator Setup

To create multiple administrators with different permission levels:

// Super Admin - Full Access
const superAdmin = new Administrator({
    adminId: 'ADM001',
    email: 'superadmin@wavemaxlaundry.com',
    permissions: ['*']  // All permissions
});

// Operations Manager - Limited Access
const opsManager = new Administrator({
    adminId: 'ADM002',
    email: 'operations@wavemaxlaundry.com',
    permissions: [
        'operator_management',
        'manage_orders',
        'view_analytics'
    ]
});

// Financial Admin - Financial Access Only
const financeAdmin = new Administrator({
    adminId: 'ADM003',
    email: 'finance@wavemaxlaundry.com',
    permissions: [
        'financial_reports',
        'view_analytics',
        'audit_logs'
    ]
});

Monitoring Administrator Activity

All administrator actions are logged. To view audit logs:

# View recent admin activity
tail -f logs/audit.log | grep "administrator"

# Check login attempts
db.administrators.find({ loginAttempts: { $gt: 0 } })

# View permission usage
db.auditlogs.find({ 
    userType: "administrator",
    createdAt: { $gte: new Date(Date.now() - 24*60*60*1000) }
})

Troubleshooting

Account Locked: After 5 failed login attempts, accounts are locked for security. To unlock: 
db.administrators.updateOne(
    { email: "admin@wavemaxlaundry.com" },
    { $set: { loginAttempts: 0, isLocked: false } }
)
Cannot Login: Verify:
  • Email address is correct (case-sensitive)
  • Account is active: isActive: true
  • No typos in password
  • Account exists in database
