1. Methodology & tools

Every finding is from open-source, vendor-neutral tools run against the live corporate site between May 19 and May 26, 2026. The tools, with the exact versions used:

  • linkinator 7.6.1 (Google's broken-link crawler) — anchor link audit, JSON output.
  • html-validate 11.4 — W3C-aligned HTML validity (severity = error, not warnings).
  • Playwright + headless Chrome 143 — full page render, console + page-error capture with stack traces, HTTP response capture for every fetched resource.
  • Google PageSpeed Insights — the public PSI API (Lighthouse scoring).
  • curl + parallel xargs — bulk HTTP-status sweeps.
  • Google's Rich Results Test — structured-data validation as Googlebot sees it (JavaScript-rendered).

All HTTP fetches with User-Agent: Mozilla/5.0. Pages audited (the four English franchise-section pages enumerated from wavemaxlaundry.com/sitemap_index.xml): /become-a-franchisee/, /laundromat-investment-guide/, /why-invest-in-wavemax/, /franchise/. The broader security and SEO findings come from prior captures with the same tool class between 2026-05-19 and 2026-05-26.

3. JavaScript crash on every load — and why the menu won't close

The franchise-conversion page throws an uncaught TypeError on every page load. Captured live in headless Chrome 143:

TypeError: $(...).on is not a function
    at https://wavemaxlaundry.com/become-a-franchisee/:2295:19

Line 2295 of the page itself contains an inline jQuery form binder:

$('#formLaundry').on('submit', function(e) { ... });

And the page loads jQuery twice — version 3.6.4 from ajax.googleapis.com at line 1655, and the WordPress-bundled 3.7.1 plus jquery-migrate at line 4200. That's a classic $-shadowing hazard that produces exactly the error above.

The operational consequence. The form-submit handlers for the Laundry and Franchise lead forms never bind. And the same broken-$ environment is the mechanism behind the user-visible behavior on the Franchise navigation: the menu's close handler — also a jQuery .on() binding in this theme — fails to attach. The menu opens; nothing closes it. The site is broken in a way the team running it could see in DevTools any time they reloaded the page.

Verify it yourself: open wavemaxlaundry.com/become-a-franchisee/, open DevTools → Console, hard-reload. Watch for the red $(...).on is not a function error referencing line 2295 of the page itself. No external scripts; the bug is in the page's own HTML.

Template-level structural defects on every page — line numbers identical across all four pages, which is how you know it's a shared (broken) template, not isolated mistakes:

7. SEO — zero local-business schema across the network

Google reads structured data (JSON-LD) on a laundromat page to decide whether the location belongs in the map pack, in "near me" results, in "laundromat in [city]" rich cards, and in the ★-rating cards. A passive rendered-DOM extraction on a 10-page sample of the corporate platform finds:

  • 9 of 10 sampled franchise pages ship only BreadcrumbList + ListItem — zero local-business schema, zero review-star markup.
  • The lone outlier (the corporate Austin page) carries 7 JSON-LD types — a remnant of CRHS's prior tuning that survived the platform switch.
  • CRHS's own page on the same Cloudflare stack ships 23 JSON-LD types including the complete local-business set (DryCleaningOrLaundry · hours · geo · services · ★ 4.8 from 48 reviews).

The 16-month Google Search Console export for www.wavemaxlaundry.com/austin-tx/ shows the consequence of shipping the platform with this gap: impressions fell from a 964/day pre-cutover average (1,427/day peak) to a 32/day post-cutover floor — a 97.7% drop, below even the original 138/day launch baseline.

Verify it yourself: point Google's Rich Results Test at any franchise page on wavemaxlaundry.com and count the "Valid items" it reports. Compare with a CRHS page on the same tool.

8. Privacy / OAuth / consumer disclosures

A passive audit of the Privacy Policy and Terms of Service linked from every franchisee footer (six locations verified) finds them missing:

  • OAuth-provider disclosures (Google / Facebook).
  • CCPA / CPRA language (California).
  • GDPR language.
  • COPPA language.
  • Retention-period statements.
  • Franchise Agreement §12.2 license-attribution markers.

These gaps affect every franchisee on the platform, not only Austin — same template, same omissions, every location.

11. The composite picture

Treated as a single corporate marketing surface — the URL franchisees are required to send prospects to — the platform delivers:

  • A franchise-conversion page that crashes on every load and 404s a network-wide link.
  • Zero structured data for the local-search results that are the product of the SEO line item the package sells.
  • Multiple OWASP-class security misconfigurations on a platform that markets "Security & Malware Protection."
  • Performance ~3× slower than the comparator built by one franchisee on the same CDN — 8× slower mobile LCP on the Austin page.
  • Zero franchise license attribution; no copyright claim in the franchisor's own content.
  • Privacy / OAuth / consumer-disclosure gaps network-wide.
  • Cross-state-identical "custom location-specific" posts on the third of franchise pages whose social feed it manages.

Every finding is measured, named, and reproducible. Run the tools yourself.
